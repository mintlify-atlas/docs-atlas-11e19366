<Note>
  SIFA is a server-rendered Django web application, not a REST API service. Nearly all functionality is delivered through HTML views rendered on the server and accessed via a browser. There is a single JSON endpoint, used internally for medication autocomplete.
</Note>

## Authentication

All routes — including the one JSON endpoint — require an active Django session. Users must log in at `/auth/login/` using their SIFA credentials. There is no public API key system, no token-based authentication, and no OAuth flow.

Once authenticated, the session cookie is attached automatically to every request by the browser. Tools that call the API programmatically (e.g. `curl`) must supply the `sessionid` cookie obtained after a successful login.

## The JSON endpoint

SIFA exposes one machine-readable endpoint:

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/productos/suggest/` | Medication autocomplete suggestions |

All other paths return HTML pages. See [Drug suggestion API](/api/products-suggest) for full documentation of this endpoint.

## Django settings that affect the API

The following `settings.py` keys control behavior of the suggest endpoint. They can be set per environment without code changes.

| Setting | Default | Description |
|---------|---------|-------------|
| `DRUG_SUGGEST_PROVIDER` | `HYBRID` | Default provider: `LOCAL`, `CIMA`, `RXNORM`, or `HYBRID` |
| `DRUG_SUGGEST_LIMIT` | `10` | Maximum number of results returned per request |
| `DRUG_SUGGEST_TIMEOUT` | `2` | Timeout in seconds for requests to external APIs |

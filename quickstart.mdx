---
title: "Quickstart"
description: "Get SIFA running in your facility"
---

This guide walks you through setting up SIFA on a local machine from scratch. By the end you will have a running development server, the four role groups configured, and a first resident with a prescription entered in the system.

<Steps>
  <Step title="Prerequisites">
    Make sure the following are installed before you begin:

    - **Python 3.10 or later** — SIFA uses Django 5.2 and requires Python 3.10+.
    - **pip** — comes bundled with Python; used to install dependencies.
    - **git** — to clone the repository.

    Verify your Python version:

    ```bash
    python --version
    # Python 3.10.x or higher
    ```
  </Step>

  <Step title="Clone the repository">
    Clone the SIFA repository and enter the project directory:

    ```bash
    git clone https://github.com/Vixoo88/Capstone_Moreno_Quintana.git
    cd "Capstone_Moreno_Quintana/Fase 2/Evidencias Proyectos/sifa_site"
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    Isolating dependencies in a virtual environment is strongly recommended:

    ```bash
    python -m venv .venv
    source .venv/bin/activate   # Windows: .venv\Scripts\activate
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages from `requirements.txt`:

    ```bash
    pip install -r requirements.txt
    ```

    Key packages that will be installed:

    | Package | Version | Purpose |
    |---|---|---|
    | `Django` | 5.2.8 | Web framework |
    | `requests` | 2.32.5 | CIMA / RxNorm / Telegram API calls |
    | `pillow` | 12.0.0 | Image handling |
    | `python-dateutil` | 2.9.0 | Date parsing utilities |
    | `sqlparse` | 0.5.3 | SQL query formatting (Django dependency) |
    | `tzdata` | 2025.2 | Timezone data |

    <Note>
      WeasyPrint is used for PDF generation but is **not** in `requirements.txt`.
      Install it separately if you need monthly medication record exports:

      ```bash
      pip install weasyprint
      ```

      SIFA will start normally without it; the PDF export route will simply be unavailable.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    SIFA reads its Telegram integration credentials from environment variables.
    Create a `.env` file in `sifa_site/` (the folder that contains `manage.py`) or export the variables in your shell:

    ```bash
    export TELEGRAM_BOT_TOKEN="your-bot-token"
    export TELEGRAM_CHAT_ID="your-chat-id"
    ```

    <Warning>
      The repository's `settings.py` includes hardcoded fallback values for development
      convenience. **Never use those values in production.** Always provide real credentials
      via environment variables before deploying.
    </Warning>
  </Step>

  <Step title="Apply database migrations">
    Create the SQLite database and apply all Django migrations:

    ```bash
    python manage.py migrate
    ```

    A `db.sqlite3` file will be created in the project root directory.
  </Step>

  <Step title="Create the superuser (Admin account)">
    Create a superuser account. This account automatically has ADMIN-level access:

    ```bash
    python manage.py createsuperuser
    ```

    Follow the prompts to set a username, email, and password.
  </Step>

  <Step title="Create role groups">
    SIFA enforces access control through four Django auth groups: **ADMIN**, **DOCTOR**,
    **TENS**, and **CUIDADORA**. Create them via the Django shell:

    ```bash
    python manage.py shell
    ```

    ```python
    from django.contrib.auth.models import Group

    for name in ["ADMIN", "DOCTOR", "TENS", "CUIDADORA"]:
        Group.objects.get_or_create(name=name)

    print("Groups created:", list(Group.objects.values_list("name", flat=True)))
    exit()
    ```

    Alternatively, create the groups through the Django Admin UI at
    `/admin/auth/group/add/` after the server is running.

    <Tip>
      Assign users to groups either in the Django Admin (`/admin/auth/user/`) or through
      the built-in user management interface at `/users/` once you are logged in.
    </Tip>
  </Step>

  <Step title="Run the development server">
    Start the Django development server:

    ```bash
    python manage.py runserver
    ```

    SIFA will be available at [http://127.0.0.1:8000](http://127.0.0.1:8000).

    Log in with the superuser credentials you created in the previous step.
    After login, Django redirects you to the dashboard at `/dashboard/`.
  </Step>

  <Step title="First actions: add a resident and create a prescription">
    With the server running and a session active, complete the following to verify everything
    is working end-to-end:

    **1. Add a resident**

    Navigate to `/residentes/nuevo/` or click **New Resident** in the sidebar.
    Fill in the resident's full name, RUT, date of birth, sex, and any known allergies.
    Save the record.

    **2. Add a medication to the catalog (optional)**

    Navigate to `/medicamentos/nuevo/` to register a drug in the local product catalog.
    This step can be skipped if you rely on the CIMA or RxNorm autocomplete instead.

    **3. Create a prescription**

    Open the resident's detail page at `/residentes/<id>/`.
    Click **New Prescription** to open the prescription form at
    `/recetas/nueva/<residente_id>/`.
    Set the start date and any observations, then save.

    **4. Add a medication order to the prescription**

    From inside the prescription, click **Add Medication Order**.
    Select a product, enter the dose, route of administration, and schedule
    (time(s) of day and optional day-of-week filter).

    **5. Review today's administration list**

    Navigate to `/administracion/` to see all doses scheduled for today.
    Mark each dose as administered, omitted, or refused.

    <Tip>
      If Telegram credentials are configured, use `/asignaciones/avisar-meds/` to send a
      low-stock alert to the facility's Telegram group.
    </Tip>
  </Step>
</Steps>
